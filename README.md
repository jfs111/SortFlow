# SORT FLOW
#### Video Demo: https://youtu.be/sYwej3gAq6U
#### Description:
SortFlow trains a K-means model using image embeddings generated by a ResNet50 backbone, based on the K classes defined by the user.
For each image — including during prediction — a trust score is computed from the distance between the image embedding and the center of its predicted cluster.

The system exposes a REST API that allows users to submit images and obtain both the predicted class and its associated trust score.

## Project Overview
The main idea behind SortFlow is to provide a tool that enables users to perform an initial classification from unlabelled images. The goal is to offer an intuitive workflow that allows people with limited technical or machine learning knowledge to benefit from automated image sorting and clustering — even when no labelled training dataset is available.

SortFlow extracts embeddings from the images, trains a clustering model, and helps users organize and understand their image collections. Once the model is built, the system also provides a simple way to integrate the prediction functionality into other applications — such as GED systems, CRMs, ERPs, and more — through a REST API.

## Features
- **Main explanatory interface** that guides users through the workflow.

- **User authentication:** account creation, login, and logout.

- **Project creation:**  
  - Requires a name, an optional description, and an estimated number of classes.  
  - Allows users to upload between **100 and 30,000 images**.  

- **Project preview:**  
  - Displays the first 50 images of the project for quick inspection.

- **Initial processing:**  
  - Extraction of embeddings using ResNet50.  
  - K-means training.  
  - Display of predicted classes and trust scores for the first 50 preview images.

- **Cluster exploration:**  
  - Visualization of each cluster with its most central image.  
  - Global trust score per cluster.  
  - Guidance for users to adjust the number of clusters based on their data.  
  - Option to refit the model with a new number of clusters **without recomputing embeddings**.

- **Cluster validation:**  
  - Images sorted by descending trust score (from 100% to 0%).  
  - Ability to exclude images and validate clusters manually.

- **Class labeling:**  
  - Users can assign a custom name to each class.

- **Class merging:**  
  - Ability to merge classes when needed.

- **Export functionality:**  
  - Export classifications in **JSON** or **CSV** format.

- **Project closure:**  
  - When closing a project, the system asks whether you want to create a new project with the rejected images.

- **API management:**  
  - Create and manage API keys.  
  - Provides sample POST requests for prediction.

- **Prediction API:**  
  - A REST POST endpoint for classification using a project ID and an API key.

- **Sample client application:**  
  - A C# example program that selects an image, encodes it in Base64, and sends a POST request to obtain the predicted class and trust score.

## Design Decisions

Several design choices in SortFlow originate from concepts learned through different machine learning specializations, combined with the practical mindset encouraged by CS50.

### 1. Inspiration from DeepLearning.AI – Siamese Networks & Embeddings
While studying the DeepLearning.AI Deep Learning Specialization, I discovered how convolutional neural networks can generate powerful embeddings that represent images beyond the traditional fully connected classification layers.

This inspired the idea of using embeddings outside supervised learning, enabling a system where unlabelled images can still be grouped meaningfully. This conceptual foundation directly shaped SortFlow.

### 2. Use of K-means – Efficient and Scalable Clustering
From the DeepLearning.AI Machine Learning Specialization, I learned the practicality and efficiency of K-means clustering, especially on high-dimensional embedding vectors.

Combining embeddings with K-means created a lightweight yet effective solution for grouping unlabelled images, making it ideal for a tool designed to be accessible to non-technical users.

### 3. Impact of the Coursera PyTorch 2024 Specialization – Power of ResNet
During the Coursera PyTorch 2024 specialization, I gained a deeper understanding of ResNet architectures and the role of skip connections.

This helped me understand why ResNet50 generalizes well to many image types, even those not seen during training, which reinforced the decision to use it as the embedding backbone of SortFlow.

### 4. Architectural Assembly – From CS50 Discipline to Personal Motivation
The overall architecture—backend, workflows, UX, API, and ML service—was made possible thanks to the engineering discipline gained through CS50.

The project was also refined during many nights of reflection (often because my one-year-old daughter kept me awake), giving me time to rethink and improve the structure. This blend of academic learning and personal experience contributed to SortFlow's final form.

## Technologies Used

### 1. Python, Flask, SQLite – Foundations from CS50
The main web service was built using Python, Flask, and SQLite, technologies directly covered during CS50. This allowed me to reuse and expand upon the knowledge gained throughout the course.

### 2. PyTorch for Deep Learning
PyTorch was chosen for embedding computation because of its clarity, strong ecosystem, and my personal preference over TensorFlow. Although I often run PyTorch models on a GPU in other projects, SortFlow was developed to run entirely on CPU, proving that the workflow remains practical even without specialized hardware.

### 3. Microservice Architecture – Separation of Responsibilities
Based on professional experience, I naturally gravitate toward modular architectures. SortFlow uses two services:

- A FastAPI service dedicated to machine learning tasks  
- A Flask service dedicated to user interaction, workflow management, and REST API exposure  

This separation allows the ML engine to be swapped for a more advanced model in the future without modifying the rest of the system.

### 4. C# Example Application – Demonstrating Interoperability
A C# Windows Forms application was developed as an example client to demonstrate interoperability with external tools and programming environments. It handles image selection, Base64 encoding, POST requests, and the interpretation of predicted classes and trust scores.

This illustrates how SortFlow can easily integrate with ERPs, CRMs, GED systems, or other enterprise applications.

## Challenges Encountered

Developing SortFlow required combining web development, machine learning, API design, database management, and multi-service orchestration. Several challenges were encountered throughout the project, each contributing to a deeper understanding of the technologies involved.

### 1. Validating the Concept
I first developed a small proof of concept using a labeled dataset to check whether the idea was viable.  
I experimented with:

- embeddings generated from ResNet50  
- K-means clustering  
- an alternative approach using DBSCAN to automatically determine the number of clusters  

DBSCAN ultimately failed for this use case due to inconsistent grouping and poor performance on high-dimensional embeddings.

However, using filenames that included class prefixes, I was pleased to observe that images from the same class were consistently grouped into the same K-means clusters.  
This confirmed that the core idea behind SortFlow was sound.

---

### 2. Building Two Interconnected Services  
One of the main challenges was designing and maintaining communication between the Flask web service and the FastAPI machine learning service.  
I had to ensure:

- reliable data transfer between services  
- a clear separation of responsibilities  
- consistent error handling  
- compatibility of data schemas (Pydantic on the ML side vs Flask forms on the web side)

This architecture required careful planning to avoid tight coupling and maintain scalability.

---

### 3. Handling Large Image Sets  
SortFlow supports up to 30,000 uploaded images per project, which introduced several technical challenges:

- the volume of data transferred between components  
- maintaining UX responsiveness during long-running processes  
- ensuring that embedding computation on CPU remained feasible  

Since the whole project is designed to run on CPU-only environments, performance tuning was essential to keep the system usable.

---

### 4. Trust Score Design  
Creating a meaningful trust score derived from clustering was not trivial. I experimented with different approaches:

- various distance metrics  
- normalization techniques  
- thresholding low-confidence images  
- ways to present the score clearly for non-technical users  

The final trust score design represents a compromise between mathematical rigor and practical usability.

---

### 5. Using AI Tools Without Losing Control of the Project  
AI tools such as Claude Sonnet assisted in generating parts of the user interface code (approximately 90% of the `sortflow_web_service`).  
However, supervising and integrating this code brought its own challenges:

- reviewing and correcting generated logic  
- adjusting architectural choices  
- ensuring consistency between both services  
- maintaining a full and deep understanding of every implemented feature  

This reinforced the need for careful engineering oversight during AI-assisted development, ensuring that all decisions remained intentional and fully under my control.

## Future Improvements

### Automatic Determination of the Optimal Number of Clusters (K)
Currently, users must provide an estimated number of classes when creating a project.  
In the future, SortFlow could automatically suggest or determine the optimal K by using:

- **Silhouette Score**: evaluating how well each image fits within its cluster compared to neighboring clusters.
- **Elbow Method**: analyzing inertia (sum of squared distances) to detect diminishing returns when increasing K.

Integrating these techniques would greatly improve usability, especially for users with no prior experience in machine learning.

## How to Run

### Prerequisites
- Python 3.10+
- 8GB+ RAM recommended
- Windows/Linux/macOS

### Installation

**1. Start the ML Service:**
```bash
cd sortflow_ml_service
pip install -r requirements.txt
python main.py
```
The ML service will start at `http://localhost:8000`

**2. Start the Web Service:**
```bash
cd sortflow_web_service
pip install -r requirements.txt
python app.py
```
The web interface will be available at `http://localhost:5000`

**3. Access the application:**
- Open your browser to `http://localhost:5000`
- Create an account and start your first project!

### API Documentation
- ML Service Swagger: `http://localhost:8000/docs`
- Web Service API: `http://localhost:5000/api/predict`

### Testing the Prediction API
Use the included C# example application (`SortFlowApiTester`) or test with cURL:
```bash
curl -X POST http://localhost:5000/api/predict \
  -H "X-API-Key: your-api-key" \
  -H "Content-Type: application/json" \
  -d '{"project_id": 1, "image_base64": "..."}'
```
